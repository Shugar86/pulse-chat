# Contributing to Pulse Chat

Спасибо, что хотите помочь! Проект развивается как pet-project, поэтому процесс максимально простой.

## Как внести изменения

1. Форкните репозиторий (если работаете не в основном).
2. Создайте ветку от `main`:
   ```bash
   git checkout -b feat/краткое-описание
   ```
3. Внесите изменения.
4. Запустите тесты и линтер:
   ```bash
   pnpm test
   pnpm lint
   ```
5. Закоммитьте с понятным сообщением:
   ```bash
   git commit -m "feat(api): добавил health check endpoint"
   ```
6. Откройте Pull Request в `main`.

## Стиль коммитов

Используем Conventional Commits:

- `feat:` — новая фича
- `fix:` — исправление бага
- `docs:` — документация
- `chore:` — рутина, обновление зависимостей
- `refactor:` — рефакторинг без изменения поведения
- `test:` — тесты

## Структура проекта

- `apps/mobile/` — React Native приложение
- `services/api/` — backend API
- `packages/shared/` — общие типы и константы
- `docs/superpowers/specs/` — дизайн-документы
- `docs/superpowers/plans/` — планы реализации

## Важно

- Следуйте существующему стилю кода.
- Минимальные изменения — чиним баг, не переписываем соседний код.
- Тесты должны проходить.
- Не коммитьте `.env` и секреты.

## Вопросы

Открывайте issue или пишите в обсуждения.
